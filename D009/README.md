# D009 — 데이터 품질 진단 및 EDA 보고서

## 데이터 개요
- **데이터**: (가상) 빵셀러 베이커리 체인 운영 데이터
- **관측 단위**: 1행 = 1건의 주문(order)
- **주요 컬럼**: 매장(store), 상품(item), 수량, 단가, 매출(revenue), 주문일시
- **분석 목적**: 매장별·상품별·월별 매출 KPI 파악

## 데이터 품질 진단
`quality_report_full()` 함수로 결측치·중복·이상치(IQR)·자료형·날짜 의심 컬럼을 한 번에 진단.

- 결측치: 60건
- 중복 행: 4건 (자연 발생 중복 포함 시 321건)
- 이상치: 음수 매출(환불) 1건
- 표기 혼재: 매장코드 공백(`" S01 "`), 상품명 이형(`"케잌"` vs `"케이크"`)
- 날짜 포맷: 3가지 혼재

## 정제 과정과 판단 근거
중복 제거 → 문자열 정제(공백/표기 통일) → 날짜 파싱 → 이상치 처리(환불 분리) → 결측치 처리 순서로 5단계 함수화, `.pipe()`로 연결.

| 단계 | 처리 내용 | 근거 |
|---|---|---|
| 1. 중복 제거 | 3건 제거 | 동일 키·값은 시스템 입력 오류로 판단 |
| 2. 문자열 정제 | 채널 4→2 통일 | `'app '`·`'APP'`은 `'app'`과 같은 채널 |
| 3. 날짜 파싱 | NaT 0건 | 월별 집계가 목적이므로 datetime 타입 필요 |
| 4. 이상치 처리 | 환불 1건 분리 + 수량 컷오프 3.0 | 매출 분석이 목적이므로 환불은 별도 보관 |
| 5. 결측치 처리 | 80건 제거 | 결제 실패 로그로 추정(MCAR 가정), 매출 왜곡 방지 |

정제 전후: **2,164행 → 1,784행**, 상품 카테고리 5종으로 정리.

## EDA 결과
- 상품별 KPI(평균 단가, 총매출, 주문건수) 집계
- **인사이트 1**: 쿠키가 매출 1위(26.69M) — 평균 단가가 아니라 **주문 건수(365건)가 가장 많아서**. 비싼 상품이 아니라 자주 팔리는 상품이 매출을 견인.
- **인사이트 2**: 케이크는 평균 단가가 가장 높지만(6,859원) 매출은 3위에 그침 — 단가와 매출 순위가 일치하지 않음.

## 한계와 후속 질문
- 이번 데이터는 3개월치만 포함되어 있어 장기적인 매출 추세나 계절성을 단정하기 어렵다.
- 결측치를 MCAR(완전 무작위 결측)로 가정하고 제거했으나, 실제로는 특정 시간대·매장에 몰려 있을 가능성을 추가로 점검할 필요가 있다.
- 다음 분석에서는 프로모션 여부, 요일, 날씨 정보를 함께 보면 매출 변화의 원인을 더 잘 설명할 수 있을 것으로 보인다.
